# 🛠 Особенности и проблемы разработки (Development Notes)

В этом документе собраны все неочевидные проблемы, баги и архитектурные особенности, с которыми мы столкнулись в процессе создания AI CRM Agent для Outlook. Документ структурирован по компонентам системы и будет крайне полезен при дальнейшей поддержке или масштабировании проекта.

---

## 1. Microsoft Outlook Add-in (Манифест и Office.js)

Интеграция с экосистемой Office оказалась самой требовательной к деталям частью проекта, особенно в контексте поддержки десктопной версии Outlook 2021.

### 1.1. Агрессивное кэширование манифеста
* **Проблема:** При обновлении файла `manifest.xml` (добавлении новых кнопок или изменении ссылок) десктопный Outlook игнорировал изменения, продолжая загружать старую версию из кэша.
* **Решение:** При каждом изменении манифеста необходимо вручную увеличивать тег `<Version>` (например, с `1.0.0.1` на `1.0.0.2`). Для полной очистки кэша в Windows иногда требуется удалять содержимое папки `%LOCALAPPDATA%\Microsoft\Office\16.0\Wef\`.

### 1.2. Валидация XML-схемы (VersionOverrides)
* **Проблема:** Outlook Desktop "молча" отказывался показывать кнопку надстройки, если структура XML хотя бы немного отклонялась от стандарта, при этом веб-версия прощала эти ошибки.
* **Решение:** Строгое соблюдение вложенности. Для поддержки старых и новых клиентов необходимо использовать базовый блок `VersionOverridesV1_0`, а внутри него (в самом конце) вкладывать `VersionOverridesV1_1`. ВАЖНО: блок `V1_1` не наследует ресурсы (картинки, тексты) из `V1_0`, их нужно полностью дублировать внутри `V1_1`.

### 1.3. Закрепление панели (SupportsPinning) и падения Outlook
* **Проблема:** При добавлении тега `<SupportsPinning>true</SupportsPinning>` (иконка "скрепки" для удержания панели открытой) Outlook 2021 выдавал критическую ошибку при переключении писем.
* **Решение:** По правилам Microsoft, если панель можно закрепить, веб-приложение *обязано* подписаться на событие смены письма. В `page.tsx` был добавлен обработчик `Office.context.mailbox.addHandlerAsync(Office.EventType.ItemChanged, ...)`, обернутый в `try-catch` для предотвращения падений.

### 1.4. Проблема с иконками (SSL и доступность)
* **Проблема:** Использование `localhost` или сервисов-заглушек (типа `placehold.co`) для иконок в манифесте приводило к ошибкам `ERR_SSL_PROTOCOL_ERROR` или блокировкам со стороны встроенного браузера Outlook.
* **Решение:** Переход на официальные стабильные CDN-ссылки Microsoft (`raw.githubusercontent.com/OfficeDev/...`) для иконок в манифесте.

## 2. Next.js и Frontend (React, Fluent UI)

### 2.1. Конфликт Office.js и Next.js Router (window.history)
* **Проблема:** При загрузке скрипта `office.js` в Next.js приложении возникала фатальная ошибка: `Application error: a client-side exception has occurred`.
* **Причина:** Скрипт `office.js` принудительно переопределяет нативные методы браузера `window.history.pushState` и `replaceState`. Next.js App Router использует эти же методы для навигации и ломается, если их сигнатура изменена.
* **Решение:** В `src/app/layout.tsx` внедрен "monkey-patch". Перед загрузкой `office.js` оригинальные методы сохраняются во временные переменные, а после загрузки — восстанавливаются обратно.

### 2.2. Интеграция Fluent UI v9
* **Проблема:** Компоненты Fluent UI требуют строгого наличия контекста темы, иначе стили не применяются или возникает ошибка гидратации (Hydration Mismatch).
* **Решение:** Создан отдельный клиентский компонент `providers.tsx` с `FluentProvider`, которым обернуто всё приложение в корневом `layout.tsx`.

## 3. Vercel AI SDK и Логика Чат-бота

### 3.1. Отрисовка вызовов инструментов (Tool Calling)
* **Проблема:** Vercel AI SDK возвращает вызовы функций (Tool Invocations) как отдельные объекты без текстового поля `content`. Из-за этого в чате появлялись "пустые пузыри" или отсутствовала индикация работы ИИ.
* **Решение:** Написана кастомная функция `renderToolInvocation`, которая перехватывает объекты инструментов и красиво отрисовывает их в виде карточек (Card) Fluent UI со статусами "Выполняется..." (Spinner) и "Завершено" (MessageBar).

### 3.2. Дублирование кнопок "Быстрых действий"
* **Проблема:** Кнопки "Привязать" и "Создать" рендерились под каждым сообщением с результатами RAG, даже если пользователь уже сделал выбор.
* **Решение:** Логика `renderQuickActions` была усложнена. Теперь она сканирует весь массив сообщений (`allMessages`) "вперед", чтобы понять, нажал ли пользователь кнопку или вызвал ли ИИ инструмент создания/привязки. Если да — старые кнопки блокируются (становятся серыми с контуром `outline`) и подсвечивают сделанный выбор зеленым цветом.

### 3.3. Визуальный разрыв сообщений ИИ
* **Проблема:** Если ИИ вызывал инструмент, а затем писал текст, это рендерилось как два отдельных сообщения подряд от "CRM Ассистент".
* **Решение:** Внедрена логика группировки (склейки) сообщений в `page.tsx`. Идущие подряд сообщения от одной роли объединяются в единый визуальный блок (чат-пузырь).

### 3.4. Промпт-инжиниринг (Борьба с галлюцинациями)
* **Проблема:** ИИ пытался выдавать markdown-ссылки на 1С, задавал лишние вопросы (просил название клиента, хотя оно было в письме) или замолкал после выполнения инструмента.
* **Решение:** Системный промпт в `route.ts` был сильно ужесточен. Добавлены прямые запреты ("НИКОГДА не выводи ссылки", "ОБЯЗАТЕЛЬНО извлеки название сам") и строгие инструкции по поведению после Tool Calling.

## 4. MCP Server и RAG (LangChain)

### 4.1. Выбор модели эмбеддингов
* **Проблема:** Изначально планировалось использовать OpenAI или локальную Ollama. OpenAI требует платного ключа, а Ollama — запущенного тяжелого локального демона, что усложняет развертывание. Google Gemini блокируется в некоторых регионах.
* **Решение:** Для LLM (текста) выбран **OpenRouter** (позволяет динамически использовать бесплатные модели). Для эмбеддингов (векторизации) выбран **HuggingFace Inference API** через LangChain — он бесплатен, не требует локальных мощностей и не имеет жестких региональных блокировок.

### 4.2. Поддержка ESM (ECMAScript Modules)
* **Проблема:** Современные версии `@modelcontextprotocol/sdk` и `langchain` поставляются только в формате ESM. При попытке скомпилировать их в CommonJS (`require`) возникали ошибки `ERR_REQUIRE_ESM`.
* **Решение:** В `mcp-1c-server/package.json` добавлено `"type": "module"`, а в `tsconfig.json` модульная система переключена на `NodeNext`.

## 5. Окружение и Инфраструктура

### 5.1. Политики выполнения PowerShell
* **Проблема:** При попытке запустить `npm install` или `npm run dev` в терминале Cursor на Windows возникала ошибка `PSSecurityException: UnauthorizedAccess`.
* **Причина:** Политика выполнения скриптов Windows (Execution Policy) по умолчанию блокирует запуск `.ps1` файлов.
* **Решение:** Использование `cmd.exe` вместо PowerShell или локальный обход политики командой `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`.
