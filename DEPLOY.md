# РУКОВОДСТВО ПО ЗАПУСКУ И РАЗВЕРТЫВАНИЮ (DEPLOY.md)

## 📌 1. Назначение и контекст
Данный документ описывает процесс локального запуска, конфигурации и развертывания проекта **AI CRM Agent for Outlook**. Система состоит из двух независимых узлов: веб-приложения на Next.js (фронтенд надстройки + ИИ-оркестратор) и локального микросервиса (MCP-сервер для интеграции с 1С и RAG-поиска).
**Целевая аудитория:** Разработчики (Frontend/Backend), системные администраторы и DevOps-инженеры, занимающиеся поддержкой и интеграцией плагина.

## 🗂 2. Связанные компоненты и файлы проекта

| Компонент / Модуль | Путь к файлу/папке в проекте | Роль в системе |
| :--- | :--- | :--- |
| **Next.js API (Оркестратор)** | `/src/app/api/chat/route.ts` | Маршрутизация запросов к OpenRouter, системный промпт, конфигурация Tool Calling. |
| **UI Чат-бота (Клиент)** | `/src/app/page.tsx` | Клиентский интерфейс на Fluent UI, извлечение данных письма через `Office.js`. |
| **Фикс роутинга (Layout)** | `/src/app/layout.tsx` | Monkey-patch для `window.history`, предотвращающий краш Next.js App Router из-за Office.js. |
| **MCP Сервер 1С** | `/mcp-1c-server/src/index.ts` | Изолированный сервис: мок-база 1С, RAG-поиск через LangChain, обработка MCP-запросов. |
| **Манифест надстройки** | `/public/manifest.xml` | XML-конфигурация для загрузки (Sideloading) плагина в Microsoft Outlook. |

## 🛠 3. Технические детали и конфигурация

### Переменные окружения (`.env.local`)
Для корректной работы системы необходимо создать файл `.env.local` в корне проекта (влияет как на Next.js, так и на MCP-сервер):
* `OPENROUTER_API_KEY` — ключ доступа к API OpenRouter. Используется в `route.ts` для вызова модели `openrouter/free`.
* `HF_TOKEN` — токен доступа к HuggingFace Inference API. Используется в `mcp-1c-server/src/index.ts` для генерации текстовых эмбеддингов (класс `HuggingFaceInferenceEmbeddings`).

### Форматы данных (Схемы)
Основная сущность в системе — **Сделка (Deal)**. В текущей реализации (в памяти MCP-сервера) она имеет следующую структуру:
```typescript
{
  id: string;          // Уникальный идентификатор (например, "1001")
  client: string;      // Название клиента (например, "ООО Ромашка")
  amount: number;      // Сумма сделки в рублях
  description: string; // Текстовое описание предмета сделки
}
```
> **Примечание:** При RAG-поиске эта структура упаковывается в объект `Document` из LangChain, где `description` попадает в `pageContent`, а остальные поля — в `metadata`.

В планах: *Замена in-memory массива `mockDeals` на реальные DTO и HTTP-запросы к OData/REST API 1С Предприятие.*

## 🚀 4. Пошаговые сценарии / Инструкции

### Сценарий 1. Запуск микросервиса интеграции (MCP Server)
Микросервис обрабатывает запросы к 1С и векторной базе. Должен быть запущен до начала работы с чатом.
1. Откройте терминал и перейдите в директорию сервера:
   ```bash
   cd mcp-1c-server
   ```
2. Установите зависимости (если запускаете впервые):
   ```bash
   npm install
   ```
3. Скомпилируйте TypeScript и запустите сервер:
   ```bash
   npm run build && node dist/index.js
   ```
   *Ожидаемый результат: В консоли появится сообщение `Vector store initialized successfully.` и `1C MCP Server running on stdio`.*

### Сценарий 2. Запуск веб-приложения (Next.js)
Обеспечивает работу UI в Outlook и проксирование запросов к LLM.
1. Откройте новый терминал в корневой папке проекта.
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите сервер разработки:
   ```bash
   npm run dev
   ```
   *Ожидаемый результат: Сервер доступен по адресу `https://localhost:3000`.*

### Сценарий 3. Установка надстройки в Outlook (Sideloading)
1. Создайте на локальном диске папку (например, `C:\OutlookManifests`) и откройте к ней общий сетевой доступ (Share).
2. Скопируйте файл `/public/manifest.xml` в эту сетевую папку.
3. В десктопном Outlook перейдите в **Файл -> Параметры -> Центр управления безопасностью -> Параметры центра управления безопасностью -> Надстройки**.
4. Установите галочку "Применить макрокоманды..." (если требуется) и перейдите в **Доверенные каталоги**.
5. Добавьте UNC-путь к вашей сетевой папке (например, `\\localhost\OutlookManifests`), поставьте галочку "Показывать в меню" и нажмите ОК.
6. Перезапустите Outlook. На вкладке "Главная" нажмите "Получить надстройки" -> "Общая папка" и выберите **AI CRM Agent**.

## ⚠️ 5. Возможные проблемы и их решение (Troubleshooting)

На основе реализованных блоков `try/catch` и логов в кодовой базе:

- **Проблема:** В консоли MCP-сервера ошибка `Failed to initialize Vector store. Did you set OPENAI_API_KEY?` (текст ошибки легаси, реально требуется HF_TOKEN).
  - **Причина:** Отсутствует переменная `HF_TOKEN` в окружении, либо нет доступа к API HuggingFace.
  - **Решение:** Проверьте наличие `HF_TOKEN` в `.env.local` и перезапустите процесс `mcp-1c-server`.

- **Проблема:** В браузере/Outlook ошибка `Chat API Error: ...` со статусом `500 Internal Server Error`.
  - **Причина:** Сбой в `route.ts`. Чаще всего вызван отсутствием `OPENROUTER_API_KEY` или временной недоступностью бесплатной модели `openrouter/free` (иногда отключается поддержка Tool Calling на стороне провайдера).
  - **Решение:** Проверьте баланс/лимиты OpenRouter или временно замените модель в `route.ts` на другую (например, `google/gemini-1.5-flash-8b`).

- **Проблема:** Outlook Desktop выдает ошибку "Это приложение невозможно установить. Файл манифеста не соответствует определению схемы".
  - **Причина:** Нарушена строгая XML-схема в `manifest.xml` (например, неправильная вложенность `VersionOverridesV1_1` или использование `SupportsPinning` в схеме 1.0).
  - **Решение:** Откатите `manifest.xml` к версии 1.0.0.8 из репозитория. Если ошибка кэшируется, увеличьте тег `<Version>` (например, до 1.0.0.9) и обновите файл в сетевой папке.

- **Проблема:** Ошибка в консоли браузера `Failed to register ItemChanged handler`.
  - **Причина:** Срабатывает `catch` блок в `page.tsx` при попытке подписаться на событие смены письма. Обычно происходит, если надстройка открыта вне контекста Outlook (в обычном браузере).
  - **Решение:** Игнорируйте эту ошибку при локальной разработке в Chrome/Edge. Внутри Outlook она перехватывается безопасно и не "роняет" приложение.
