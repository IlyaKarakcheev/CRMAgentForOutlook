# 🗺️ ROADMAP РАЗРАБОТКИ: AI CRM AGENT ДЛЯ OUTLOOK

## 📌 1. Назначение документа
Данный Roadmap (дорожная карта) описывает поэтапный план разработки интеллектуальной надстройки для Microsoft Outlook с интеграцией CRM (например, 1С) через протокол MCP и использованием ИИ (Vercel AI SDK + RAG). 
План составлен с учетом реальных архитектурных вызовов, ограничений экосистемы Office.js и требований к UX/UI.

**Целевая аудитория:** Product Managers, Tech Leads, разработчики и стейкхолдеры проекта.

---

## 📍 Фаза 1: Базовая инфраструктура и каркас надстройки (Недели 1-2)
**Цель:** Создать фундамент веб-приложения и успешно загрузить "пустой" плагин в десктопный и веб-клиент Outlook.

* **Шаг 1.1: Инициализация Frontend-ядра**
  * Развертывание проекта на Next.js (App Router).
  * Интеграция **Fluent UI React v9** (настройка `FluentProvider` для соответствия дизайн-коду Microsoft).
* **Шаг 1.2: Настройка Office.js**
  * Подключение библиотеки `office.js`.
  * Реализация monkey-patching для `window.history` (критически важно для предотвращения крашей Next.js роутера).
* **Шаг 1.3: Разработка Манифеста (XML)**
  * Создание `manifest.xml` со строгим соблюдением схем `VersionOverridesV1_0` и `VersionOverridesV1_1`.
  * Настройка HTTPS для локальной разработки.
  * Размещение статичных ресурсов (иконок) на надежном CDN для обхода блокировок десктопного Outlook.
* **Шаг 1.4: Sideloading и тестирование**
  * Загрузка манифеста через сетевую папку (Share) в десктопный Outlook 2021/365.
  * Настройка извлечения базовых данных письма (Отправитель, Тема, Тело письма).

---

## 📍 Фаза 2: Интеграционный слой и RAG-система (Недели 3-4)
**Цель:** Создать изолированный микросервис (MCP Server), который будет отвечать за логику CRM и семантический поиск.

* **Шаг 2.1: Инициализация MCP-сервера**
  * Создание отдельного Node.js сервиса с поддержкой ESM.
  * Подключение `@modelcontextprotocol/sdk` (Stdio transport).
* **Шаг 2.2: Реализация базовых инструментов (Tooling)**
  * Разработка методов: `search_deal_exact`, `create_deal`, `link_deal`, `update_deal_amount`, `delete_deal`.
  * Подключение моковой БД (на этапе PoC) или разработка коннекторов к REST/OData API реальной CRM.
* **Шаг 2.3: Построение RAG-конвейера (Семантический поиск)**
  * Интеграция `LangChain`.
  * Подключение провайдера эмбеддингов (например, HuggingFace Inference API).
  * Настройка векторного хранилища (`MemoryVectorStore` для PoC, с заделом под PgVector/Pinecone).
  * Реализация инструмента `search_similar_deals_rag`.

---

## 📍 Фаза 3: ИИ-Оркестрация и Промпт-инжиниринг (Недели 5-6)
**Цель:** Связать фронтенд с LLM, настроить логику принятия решений и вызов инструментов.

* **Шаг 3.1: Интеграция Vercel AI SDK**
  * Создание API Route (`/api/chat/route.ts`).
  * Настройка потоковой передачи ответов (Streaming) через `useChat`.
* **Шаг 3.2: Подключение LLM Провайдера**
  * Настройка подключения к OpenRouter (или напрямую к OpenAI/Azure OpenAI).
  * Активация режима `maxSteps` для поддержки цепочек вызова инструментов (Tool Calling).
* **Шаг 3.3: Промпт-инжиниринг**
  * Разработка строгого системного промпта.
  * Настройка правил поведения: запрет на генерацию markdown-ссылок, обязательный запрос недостающих данных у пользователя, правила форматирования финальных ответов.

---

## 📍 Фаза 4: UX/UI и Human-in-the-Loop (Недели 7-8)
**Цель:** Сделать интерфейс интуитивно понятным, безопасным и визуально приятным для конечного пользователя.

* **Шаг 4.1: Визуализация работы ИИ**
  * Разработка карточек (Cards) для отображения статуса инструментов ("Выполняется...", "Завершено").
  * Склейка (группировка) идущих подряд сообщений от ИИ в единый визуальный блок.
* **Шаг 4.2: Интерактивные быстрые действия (Кнопки)**
  * Динамическая генерация кнопок ("Привязать к [ID]", "Создать новую") на основе ответов RAG.
  * Реализация логики "заглядывания в будущее": блокировка кнопок (серый контур) и подсветка выбранного действия (зеленый фон) при навигации по истории чата.
* **Шаг 4.3: Улучшение эргономики Outlook**
  * Включение функции закрепления панели (`SupportsPinning` в манифесте).
  * Добавление обработчика `ItemChanged` для безопасного переключения между письмами без падения плагина.
  * Автоматический скролл чата к новым сообщениям.

---

## 📍 Фаза 5: Production, Безопасность и Релиз (Недели 9-10)
**Цель:** Подготовить проект к развертыванию в реальной корпоративной среде.

* **Шаг 5.1: Аутентификация и Авторизация (SSO)**
  * Интеграция Microsoft Entra ID (Azure AD) для получения токена текущего пользователя Outlook.
  * Проброс токена в MCP-сервер для авторизации запросов в 1С от имени конкретного сотрудника.
* **Шаг 5.2: Переход на Production Инфраструктуру**
  * Замена `MemoryVectorStore` на персистентную векторную БД (например, Qdrant или PostgreSQL + pgvector).
  * Замена моковых данных на реальные шлюзы к 1С Предприятие.
* **Шаг 5.3: CI/CD и Деплой**
  * Настройка пайплайнов сборки (GitHub Actions / GitLab CI).
  * Развертывание Next.js приложения на корпоративных серверах (Docker) или в облаке.
* **Шаг 5.4: Централизованное развертывание в Microsoft 365**
  * Публикация `manifest.xml` через Центр администрирования Microsoft 365 (Centralized Deployment) для автоматической установки плагина всем сотрудникам отдела продаж.

## 📈 Метрики успеха проекта (KPIs)
1. **Время обработки письма:** Снижение времени на занесение данных в CRM с 3-5 минут до 15-30 секунд.
2. **Точность RAG:** Процент успешных нахождений нужной сделки по смыслу письма > 85%.
3. **Стабильность (Crash-rate):** Отсутствие падений панели в десктопном Outlook при переключении писем.
4. **Adoption rate:** Процент менеджеров, регулярно использующих плагин вместо ручного переключения в окно 1С.
