Подробное описание разработки: AI CRM Agent для Outlook
1. Обзор проекта
Цель проекта: Создание интеллектуальной надстройки (Add-in) для Microsoft Outlook, которая позволяет менеджеру обрабатывать входящие письма и управлять сделками в 1С CRM прямо из интерфейса почтового клиента, не переключаясь между окнами.

Основной сценарий (Workflow):

Пользователь открывает письмо в Outlook и запускает панель ИИ-ассистента.
ИИ анализирует отправителя, тему и текст письма.
ИИ классифицирует письмо (спам, личное, рабочее) и пытается найти соответствующую сделку в 1С с помощью точного поиска.
Если точного совпадения нет, ИИ использует семантический поиск (RAG), чтобы найти похожие сделки по смыслу текста.
ИИ предлагает пользователю выбор (в виде кнопок): привязать письмо к одной из найденных сделок или создать новую.
Взаимодействие происходит по принципу Human-in-the-loop (человек в контуре): ИИ не делает деструктивных или создающих действий без прямого клика пользователя.
Поддерживается полный цикл работы со сделкой: создание, привязка, изменение суммы и удаление.
2. Архитектура и Технологический стек
Проект разделен на две независимые части, взаимодействующие по протоколу MCP (Model Context Protocol).

Фронтенд и Бэкенд оркестратора (Next.js)
Фреймворк: Next.js (App Router) + React.
UI Библиотека: Fluent UI React v9 (официальная дизайн-система Microsoft для 100% визуальной интеграции с Outlook и Teams).
AI Оркестратор: Vercel AI SDK (ai, @ai-sdk/openai). Отвечает за потоковую передачу ответов (streaming), управление состоянием чата и вызов инструментов (Tool Calling).
LLM Провайдер: OpenRouter (openrouter/free). Позволяет использовать современные LLM бесплатно и без региональных блокировок.
Интеграция с почтой: Office.js (Microsoft Office JavaScript API) — для чтения контекста открытого письма.
Микросервис интеграции с 1С (MCP Server)
Протокол: Model Context Protocol (MCP) через Stdio.
Стек: Node.js, TypeScript.
Семантический поиск (RAG):
Оркестратор: LangChain (@langchain/core).
Векторная БД: MemoryVectorStore (БД в оперативной памяти для демо).
Модель эмбеддингов: HuggingFace Inference API (@langchain/community).
3. Основные реализованные функции (Фичи)
🧩 Умный контекстный UI (Human-in-the-loop)
Вместо того чтобы заставлять пользователя печатать команды, интерфейс динамически генерирует кнопки "Быстрых действий" («Привязать к [Сделка]», «Создать новую»).

Кнопки появляются только тогда, когда ИИ находит похожие сделки с помощью RAG.
После нажатия выбранная кнопка подсвечивается зеленым и остается в истории чата, а остальные варианты блокируются (серый контур), создавая понятную визуальную историю решений.
Сообщения ИИ и результаты вызова инструментов склеиваются в единые визуальные блоки для удобства чтения.
🛠️ Tool Calling (Вызов функций)
ИИ-ассистент обучен вызывать строго определенные инструменты (функции) в 1С:

search_deal_exact — быстрый поиск по ID или точному названию компании.
search_similar_deals_rag — "умный" поиск сделок по контексту письма.
create_deal — создание новой сделки с авто-извлечением описания и названия из письма.
link_deal — привязка текущего письма к существующей сделке.
update_deal_amount — корректировка суммы сделки.
delete_deal — удаление/отмена сделки.
🔍 Семантический RAG-поиск
Если в письме написано "обновление дизайна сайта", а в базе 1С сделка называется "Разработка корпоративного портала", обычный текстовый поиск не сработает. RAG-система превращает тексты в числовые векторы (эмбеддинги) через HuggingFace и находит смысловые совпадения с помощью вычисления косинусного расстояния.

📌 Глубокая интеграция с Outlook
Закрепление панели (Pinning): Настроена поддержка SupportsPinning в манифесте версии 1.1. Панель можно закрепить "скрепкой", и она не будет закрываться при переключении писем.
Перехват событий: Настроен слушатель ItemChanged, который позволяет реагировать на смену писем при закрепленной панели.
Изоляция роутинга: Реализован обходной путь (monkey-patch) для сохранения методов window.history, чтобы Next.js Router не ломался из-за особенностей работы среды Office.js.
4. Этапы разработки (Хронология)
Инициализация и настройка среды:
Развернут базовый проект Next.js.
Настроен Microsoft Outlook Manifest (manifest.xml) для загрузки (sideloading) приложения в десктопный и веб-клиент Outlook.
Переход на Fluent UI:
Полный отказ от стандартного HTML/Tailwind в пользу @fluentui/react-components, чтобы плагин выглядел как родная часть Office 365.
Разработка MCP-сервера "1С":
Создан отдельный Node.js процесс.
Написаны моковые данные сделок.
Интегрирован LangChain и векторная база для RAG.
Описаны схемы (InputSchema) для инструментов по стандарту MCP.
Оркестрация ИИ:
Настроен Vercel AI SDK.
Сформирован строгий системный промпт, который запрещает ИИ самовольничать (например, запрещено выдавать markdown-ссылки, запрещено создавать сделки без подтверждения пользователя, обязательно спрашивать недостающие данные).
Полировка UX/UI:
Добавлен автоматический скролл вниз при появлении новых сообщений.
Стилизованы карточки инструментов (Tool Cards) с иконками загрузки и статусом.
Написана сложная логика "заглядывания в будущее" для кнопок, чтобы они правильно окрашивались в зеленый цвет при навигации по истории чата.
Решение проблем совместимости Outlook Desktop:
Пройдена сложная отладка XML-схем (VersionOverridesV1_0 и V1_1), подбор правильных CDN-иконок и настройка кэширования для работы кнопки на десктопной версии Outlook 2021.
5. Как запустить систему для разработки
Для работы системы требуются два параллельно запущенных процесса.

Шаг 1. Запуск микросервиса 1С (MCP Server) Откройте первый терминал в папке проекта:

cd mcp-1c-server
npm run build && node dist/index.js
Этот сервер будет висеть и ждать команд по протоколу Stdio.

Шаг 2. Запуск фронтенда (Next.js) Откройте второй терминал в корневой папке проекта:

npm run dev
Сервер запустится на https://localhost:3000.

Шаг 3. Загрузка в Outlook

Положите файл public/manifest.xml в расшаренную сетевую папку (Share).
В Outlook перейдите в раздел надстроек и добавьте манифест из общей папки.
Откройте любое письмо, нажмите появившуюся кнопку "Process Email" на панели сверху.
